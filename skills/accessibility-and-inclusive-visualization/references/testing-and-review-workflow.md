# Testing and Review Workflow

## What Problem This Solves

This reference gives the skill a repeatable review flow for accessible charts and dashboards.

## When to Use It

Use this when auditing a visual before release or export.

## Key Takeaways

- Review contrast, redundant encoding, focus, hover dependence, text alternatives, and exported fallback paths.
- Test the surrounding narrative and layout, not just the chart region.
- Accessibility review should happen while the chart is still changeable.

## Common Mistakes

- Reviewing only the default state and not the interactive states.
- Forgetting document exports and screenshots.
