# Vega-Lite and Vega

## What Problem This Solves

This reference covers the most portable declarative path for browser-ready specs.

## When to Use It

Use this when you need layered, transformed, faceted, or embedded specs that should survive across notebooks, web apps, or export workflows.

## Key Takeaways

- Vega-Lite is best when high-level marks, transforms, and composition cover the job cleanly.
- Vega is the fallback when lower-level control is needed while keeping a declarative runtime.
- Specs are easier to validate, store, and transport than bespoke rendering code.

## Common Mistakes

- Forcing a chart into Vega-Lite after the spec becomes harder to read than custom code.
- Ignoring accessibility and text alternatives just because the chart is declarative.
