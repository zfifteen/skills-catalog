1) Core data structures
Metadata Index: A transactional database mapping logical file paths or object IDs to their content-addressed hashes.
Blob Store: An append-only storage layer (e.g., S3 or a local filesystem) keyed by cryptographic hash (e.g., SHA-256). Crucially, the storage metadata must track the creation_timestamp of every blob.
ReferencedSet: An in-memory hash set or Bloom filter populated during the GC run to store all hashes currently linked by the Metadata Index.
